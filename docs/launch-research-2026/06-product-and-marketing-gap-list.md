# 06 Product and Marketing Gap List

Last updated: 2026-06-03, Asia/Colombo.

This is a launch backlog, not an implementation change. The current task did not change app code.

## Critical Before Public Launch

1. Decide final public domain.
   - Repo and production references say `dinaya.lk`.
   - Existing 30-second voiceover script says `dinaya.app`.
   - Pick one before publishing video/post.

2. Prepare 3 demo accounts/pages.
   - Salon/wellness.
   - Clinic/consultant.
   - Fitness/classes.
   - Each demo should show real services, prices, staff, availability, booking page, and dashboard.

3. Prepare founder setup intake.
   - Could be Google Form, Typeform, or direct WhatsApp template.
   - Must collect services, durations, prices, staff, hours, location, logo, phone, and policies.

4. Prepare tracking sheet.
   - Lead, channel, UTM, referral code, owner, vertical, stage, notes, next follow-up, source, activation, paid status.

5. Prepare launch FAQ.
   - Pricing.
   - Trial.
   - PayHere.
   - Data/privacy.
   - WhatsApp/SMS.
   - Setup time.
   - What happens after a booking.
   - AI Voice status.

## Product Gaps

### Promo Codes

Status:

- Not coded as a true product flow.

Impact:

- Can run manual launch codes, but cannot honestly market in-app automatic promo redemption yet.

Recommended priority:

- High after launch if referrals/creator campaigns show demand.

Build later:

- Subscription promo codes for Dinaya plans.
- Tenant booking promo codes for customers.
- Affiliate/creator codes for acquisition.
- Admin reporting and anti-abuse.

### Waitlist or Founder Setup Funnel

Status:

- Product has registration and public pages, but launch may need a simpler "free setup" capture path.

Impact:

- Cold traffic from videos may not want to register immediately.

No-code workaround:

- Use a form and WhatsApp CTA for the first 30-50 founder setups.

Build later:

- Dedicated founder setup page.
- Intake form connected to admin.
- Auto-create draft business setup tasks.

### Case Studies

Status:

- Needed for trust.

Impact:

- Sri Lankan SMEs may need proof from businesses that look like them.

Workaround:

- Use simple text/video testimonials from pilots.

Build later:

- Public customer stories page.

### Sinhala and Tamil Launch Support

Status:

- Public competitor messaging includes multilingual access in places.

Impact:

- English-only launch can work for Colombo/startup audiences, but broader SME adoption benefits from Sinhala/Tamil support.

Workaround:

- Create Sinhala/Tamil social captions, WhatsApp templates, and onboarding support manually.

Build later:

- Localized booking pages and dashboard strings if not already complete.

### Influencer/Affiliate Dashboard

Status:

- Referral links exist, but influencer payout workflow is not a full product/admin feature.

Impact:

- Manual tracking works for first partners, but will break if scaled too fast.

Workaround:

- Track creator links/codes in a sheet and pay only after verified paid conversion.

Build later:

- Partner table, payout reports, attribution windows, and fraud controls.

### Launch Analytics

Status:

- GA4 is optional per deployment docs for public deal funnel analytics.

Impact:

- Launch decisions need source and activation data.

Workaround:

- Use UTMs, referral codes, manual sheet, and dashboard metrics.

Build later:

- Acquisition source dashboard for platform admins.
- Funnel view: visitor -> signup -> setup -> first booking -> paid.

### AI Voice Receptionist

Status:

- Coming soon. Tenant setup requests and voice API flow are paused until rollout.

Impact:

- Strong marketing angle, but dangerous to overpromise.

Recommendation:

- Mention as "coming soon for Growth/managed setup" only when appropriate.
- Do not make it a primary launch CTA.

## Marketing Gaps

1. Clear launch domain and CTA.
   - Decide whether the CTA is "Try free", "Get free setup", or "DM DINAYA."
   - For launch, "Get free setup" is strongest.

2. Vertical landing narratives.
   - "Dinaya for salons and spas."
   - "Dinaya for clinics."
   - "Dinaya for fitness studios."
   - These can start as social posts before becoming pages.

3. Proof library.
   - First booking screenshot.
   - First business quote.
   - First Deal conversion.
   - First PayHere deposit.
   - First QR share.

4. Objection content.
   - "We already use WhatsApp."
   - "My customers call."
   - "I do not need online payments."
   - "Will older customers use this?"
   - "What if someone books wrong?"

5. Partner kit.
   - One-pager for creators/agencies.
   - Tracked link/code.
   - Script and visual brief.
   - Terms and payout rules.
