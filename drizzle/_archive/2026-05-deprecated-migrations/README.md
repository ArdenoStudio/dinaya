# 2026-05 — Deprecated migrations

## What was here

Five drizzle-kit artifacts that lived locally (never committed to git) before
Suven re-sequenced the migration set in commit `00127774`. Removed on
**2026-05-20**.

| File                                | Status when removed | Replaced by                                                |
| ----------------------------------- | ------------------- | ---------------------------------------------------------- |
| `drizzle/0000_grey_nova.sql`        | Applied to Neon     | Subsumed by `drizzle/0001_clients_crm.sql` + `0002_s1_foundations.sql` |
| `drizzle/0002_high_thanos.sql`      | Applied to Neon     | `drizzle/0002_s1_foundations.sql`                          |
| `drizzle/0003_remove_dodo.sql`      | **Never applied**   | `drizzle/0003_platform_stabilization.sql`                  |
| `drizzle/0004_profile_and_reviews.sql` | **Never applied** | `drizzle/0004_sri_lanka_growth.sql`                        |
| `drizzle/meta/` (folder)            | Local-only cache    | Regenerated by `drizzle-kit generate`                      |

## Why

The first two had run against the dev Neon DB during early scaffolding. Suven
later normalised the migration sequence into the canonical `0001`–`0004` set
committed in `00127774`. The matching numeric slots (`0003`, `0004`) were then
re-used for different content, which would have caused drizzle-kit to refuse
to apply or to apply the wrong file depending on disk order.

The DB's `drizzle.__drizzle_migrations` table (queried 2026-05-20) confirms
that whatever ran has been superseded by Suven's sequence; the live schema
matches `0001`–`0004` as currently committed.

## File contents

**Not preserved.** These files were deleted before the archive convention
existed (this folder was created reactively). They were untracked locally and
never committed to git or captured by `git stash -u`, so the bytes are
unrecoverable.

If you ever need to reconstruct what `0003_remove_dodo.sql` did, the closest
reference is the deletion of `src/app/api/webhooks/dodo/route.ts` in commit
`76ba855c` plus the related references that Suven cleaned up in
`00127774`.

## Lesson

Going forward: **archive before delete**, even when files look obviously
disposable. The Drizzle Migrations Archive convention (see
`../README.md`) is the result of this incident.
